penjelasan dan fungsi

git init 
git status
git commit
git push origin main

1.git init
Penjelasan: Perintah ini digunakan untuk membuat repositori Git baru.
Fungsi: Menginisialisasi folder proyek kamu agar mulai dilacak oleh Git. Setelah menjalankan perintah ini, sebuah folder tersembunyi bernama .gitakan dibuat di dalam direktori proyekmu

2.git status
Penjelasan: Memberikan informasi mengenai kondisi terkini dari direktori kerja (folder kerja) dan staging area .
Fungsi: Menunjukkan file mana saja yang telah diubah, file mana yang sudah siap untuk di-commit ( staged ), dan file mana yang belum dilacak oleh Git ( untracked )

3.git commit
Penjelasan: mengambil "foto" ( snapshot ) dari perubahan yang telah kamu siapkan di staging area .
Fungsi: menyimpan perubahan secara permanen ke dalam riwayat (history) Git lokal kamu. Biasanya digunakan dengan flag -muntuk memberikan pesan singkat mengenai perubahan yang dibuat, contoh: git commit -m "menambah fitur login"

4.git push origin main
Penjelasan: Mengirimkan commit yang ada di repositori lokal ke repositori jarak jauh ( remote repositori ).
Fungsi:
push: Perintah untuk mengirim data.
origin: Nama alias untuk repositori URL jarak jauh (misalnya di GitHub atau GitLab).
main: Nama cabang (cabang) tujuan tempat kode kamu akan disimpan.

5.git add
git addadalah perintah yang digunakan untuk memindahkan perubahan dari direktori kerja (folder kerja) ke Staging Area (juga dikenal sebagai "Indeks")
fungsi : Menyeleksi Perubahan: Kamu tidak harus menyimpan semua perubahan sekaligus. Dengan git add, kamu bisa memilih file spesifik mana saja yang sudah siap untuk di- commit
